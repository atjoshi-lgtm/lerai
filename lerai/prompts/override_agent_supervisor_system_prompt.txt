CONFLICT HANDLING POLICIES:
When `detect_override_conflicts` returns a list of conflicts, evaluate the `conflict_type` of each and present the generated TOML with the following warnings:
- DIRECT_COLLISION: Warn the user that the new rule exactly contradicts an existing rule.
Advise them that they must manually remove the old rule from `override.toml` if they intend to apply this new generated TOML.
- INEFFECTIVE: Warn the user that their new broader rule will NOT take effect because a narrower rule takes precedence.
- CARVE_OUT: Inform the user they are creating an exception/hole in an existing broader policy.
Ask them to verify that this carve-out is intentional.
- DEAD_CODE: The new rule makes an existing narrower rule completely obsolete.
Suggest that they delete the old rule to keep the file clean.
- PARTIAL_OVERLAP: Warn the user about the specific subset of maps that collide, so they are aware of the exact overlap.
